Angular Datagrid
for AngularJS

Built for handling VERY LARGE LISTS in Angular and making them initialize quickly and scroll smoothly.

Datagrid for Angular JS no longer limits you to having to paginate your data. You can now load HUGE lists (Tested with 50,000 Rows!) and have your app scroll like a champ!

Getting Started / Setup

Start by adding the angular-ux-datagrid.js to your project. You can download it by right clicking this link and saving the file.


    <script src="angular-ux-datagrid.js"></script>

Then adding the ux to your app module. This registers the datagrid directive.


    var app = angular.module('app', ['ux']);

Now we need to setup a controller with some items for us to display in our datagrid.


    angular.module('app').controller('myCtrl', function($scope) {
        var items = [];
        for (var i = 0; i < 100; i += 1) {
            items.push({id:i});
        }
        $scope.items = items;
    });
Default Recommended CSS

The default CSS that every grid should have. This makes rows fade in after they render. And hides them until they are rendered.

NOTE:To make rows fade in correctly each row must have it's content in at least one child element. For example if I have a row that has a name in it and the row is a div. Then the row should have a span, and then the name. This allows the css to fade it in without hiding the entire row, because it only hides the contents of a row until it is rendered.


.datagrid-row > * {
   -webkit-transition: opacity 0.5s ease-out;
   -moz-transition: opacity 0.5s ease-out;
   -ms-transition: opacity 0.5s ease-out;
   -o-transition: opacity 0.5s ease-out;
   transition: opacity 0.5s ease-out;
}

.uncompiled > * {
   visibility: hidden;
   opacity: 0;
}

.compiled > * {
   opacity: 1;
}

.datagrid-chunk {
   opacity: 0;
   -webkit-transition: opacity 0.5s ease-out;
   -moz-transition: opacity 0.5s ease-out;
   -ms-transition: opacity 0.5s ease-out;
   -o-transition: opacity 0.5s ease-out;
   transition: opacity 0.5s ease-out;
}

.datagrid-chunk-ready {
   opacity: 1;
}
A Simple Datagrid

Start by adding the ux-datagrid on a DOM container element. (Typically this will be a DIV).

Take special note to the script tags. The templates are contained in them. This keeps the browser from evaluating the html inside and allows us auto complete of html is most editors.

The template-name sets the name of that template if you don't specify one, the name becomes default. If you have two with the same or no name, they will overwrite each other.

The template-item sets the name by which you want to reference the data on your scope for that row. (ex 'item' == $scope.item).

  • HTML
  • CSS
  • JavaScript

<div ng-controller="ctrl">
    <div ux-datagrid="items">
        <script type="template/html" template-item="item">
            <div class="ex1-row"><span>{{item.id}}</span></div>
        </script>
    </div>
</div>


.ex1 *[ux-datagrid] {
    width: 100%;
    height: 150px;
}

.ex1-row {
    background-color:#efefef;
    height: 20px;
    line-height: 20px;
    white-space: nowrap;
    text-overflow: ellipsis;
    overflow: hidden;
    margin-top: 5px;
}
.ex1-row:nth-child(odd) {
    background-color:#dfdfdf;
}


var name = 'ex1';
angular.module(name, ['ux'])
    .controller('ctrl', function ($scope) {
        var i = 0, len = 100, items = [];
        while (i < len) {
            items.push({id: i});
            i += 1;
        }
        $scope.items = items;
    });

The datagrid will then need you to pass some data to it. The simplest way it so set an array on your $scope.

Using Addons

Addons allow you to change the behavior or add features to the datagrid. First you need to include the addon that you want. Let's start with ScrollBar.


    <script src="ux-datagrid-scrollBar.js"></script>

Now we need add the addon to the grid that we are using. We do this by adding an addons attribute to the html.


    <div ux-datagrid="items" addons="scrollBar">...

That's really it. You can add as many addons to a grid as you like.


    <div ux-datagrid="items" addons="scrollBar scrollBounce">...
Using Options

Options configure different parts or the datagrid, or configure an addon. The datagrid comes with a list of global options. Those are on ux.datagrid.options. When you add options to a grid gets a copy of the global options. Any options that you pass to the grid then override or add to that options object on that grid only.


    <div ux-datagrid="items" options="{smartUpdate:false}">...

So if you want to change the behavior for all grids, then you can modify the global options by just overriding them directly on the ux.datagrid.options.


    ux.datgrid.options.smartUpdate = false;
Using Templates

Templates templates are by default built into script tags for convenience. While this works good for your dev environment, your production environment should really be using something like a grunt for ng-templates that will minimize your html and put it into your script so you can just load your js files.

The datagrid also supports passing in your templates instead of defining them in script tags. 


    // you can do as many templates as you like just like the script templates.
    // we just need them assigned on some variable on the scope, so they can be pulled by the options later.
    $scope.templates = [
        {
            //You could use $templateCache['templateName'] here.
            template: '<div class="row"><div>{{item.id}}</div></div>',
            name: 'default',
            item: 'item',
            base: null
        }
    ];

So if you want to change the behavior for all grids, then you can modify the global options by just overriding them directly on the ux.datagrid.options.


    <div ux-datagrid="items" options="{templateModel:{templates:templates}}"></div>
Build Your First Addon

Let's first set the stage. You have a grid, and you want to make it show different templates for each row. The first part we need is to know how to tell what item gets what template. So we will make an addon to do this.

Let's also assume for this example that your data looks like this. You have a type property on most of your objects that you are using to tell them apart.


$scope.items = [
        {id:0},
        {id:0, type: 'alt1'},
        {id:1, type: 'alt2'},
    ];

Given that data, we want to have different rows for each type. So we will setup our addon to override the names so it will use those type values.


angular.module('ux').factory('whichTemplate', function () {
    return ['inst', function (inst) {
        // now we override it with our method so we decide what template gets displayed for each row.
        inst.templateModel.getTemplate = function (item) {
            // item would be the item in your array of data provided to the datagrid.
            // let's make our default option be 'row'. Otherwise datagrid defaults this to 'default'.
            var name = item.type || 'row';
            // now we get the template from the name.
            return inst.templateModel.getTemplateByName(name);
        };
    }];
});

Now we just need to make sure we have templates for each of those types.


<div ng-controller="ctrl">
    <div ux-datagrid="items">
        <script type="template/html" template-item="item">
            <div class="ex2-row"><span>{{item.id}}</span></div>
        </script>
        <script type="template/html" template-name="alt1" template-item="item">
            <div class="ex2-row alt1"><span>ALT1: {{item.id}}</span></div>
        </script>
        <script type="template/html" template-name="alt2" template-item="item">
            <div class="ex2-row alt2"><span>This {{item.id}} is ALT2</span></div>
        </script>
    </div>
</div>

    </div>
</div>

var name = 'grouped';
angular.module(name, ['ux'])
    .controller('ctrl', function ($scope) {
        // we are going to create some grouped data. Array of items with .children arrays of items.
        // top level items become the group headers, the children items become the rows.
        var i = 0, j, len = 10, items = [], item, childLen = 0;
        while (i < len) {
            item = {id: i, children: []};
            items.push(item);
            childLen = Math.random() * 10;
            j = 0;
            while (j < childLen) {
                item.children.push({id: i + '.' + j});
                j += 1;
            }
            i += 1;
        }
        $scope.items = items;
    });
angular.module('ux').factory('whichTemplate', function () {
     return ['inst', function (inst) {
        // now we override it with our method so we decide what template gets displayed for each row.
        inst.templateModel.getTemplate = function (item) {
            // item would be the item in your array of data provided to the datagrid.
            // let's make our default option be 'row'. Otherwise datagrid defaults this to 'default'.
            var name = item.children ? 'group' : 'row';
            // now we get the template from the name.
            return inst.templateModel.getTemplateByName(name);
        };
    }];
});
        strapped = true;

Resize Columned Data (like tables)

Allow columns to be resized. This cannot be done with a grid addon because it needs to affect both the elements of the grid and elements outside of it (the header columns).

There are a few specific rules for this to work. First of all you must provide a selector that will match both the header rows and the grid rows.

This selector will then be used to create dynamic style rules so it will affect all children. Putting a style tag on each row wouldn't work because they may not all be there (because of chunking), and it would be super slow.

NOTE: if you do not make your selector specific enough it will not override your current styles.

  • HTML
  • CSS
  • JavaScript

        <div class="col col3">Last Name</div>
        <div class="col col4">Age</div>
        <div class="col col5">Kids</div>

angular.module('ux').factory('whichTemplate', function () {
     return ['inst', function (inst) {
        // now we override it with our method so we decide what template gets displayed for each row.
        inst.templateModel.getTemplate = function (item) {
            // item would be the item in your array of data provided to the datagrid.
            // let's make our default option be 'row'. Otherwise datagrid defaults this to 'default'.

            // now we get the template from the name.
            return inst.templateModel.getTemplateByName(name);
        };
    }];

There are different options for the logging to help filter it down. {debug:{all:1, Flow:0}} is what I use most of the time. It gives you all of the logs except the Flow control. If the internal flow control logs are turned on, then you see it's internal handling of methods to make sure they are fired in the correct order. I don't think most will care to use that.. It can be quite verbose.

But what you notice from this is that you can turn the logs on and off for any particular module just by listing it's name and giving it a 1 or a 0. If you wanted to turn off logs for the templateModel then you would add templateModel:0 to the options of the debug.

You could try using this set of options below to turn off the events. {debug:{all:1, Flow:0, events:0}}.

 function makeStr(len) {
    var str = '';
    while(str.length < len) {
